OUR STORY

Why We Exist

WHY TO
FASHION?

Why To Fashion (WTF) was born from a simple frustration: great menswear in Sri Lanka was either overpriced imported brands or generic local alternatives with no character.

Founded in Vavuniya, we set out to create a brand that speaks to young Sri Lankan men — bold enough to make a statement, accessible enough to wear every day.

Every piece in our collection is handpicked for quality, fit, and attitude. Because we believe the way you dress is a direct expression of who you are.

M

OUR MISSION

To provide quality, stylish menswear that empowers young men to express themselves boldly — without breaking the bank. We're here to prove that style has no price tag.

V

OUR VISION

To become the defining menswear brand of Sri Lanka — a label that young men wear with pride, not just for the clothes, but for what the brand stands for: authenticity and confidence.

SUPABASE SETUP

1. Create Tables in Supabase SQL Editor

-- Products table
CREATE TABLE products (
  id uuid DEFAULT gen_random_uuid() PRIMARY KEY,
  name text NOT NULL,
  price numeric NOT NULL,
  image_url text,
  description text,
  category text,
  created_at timestamptz DEFAULT now()
);

-- Reviews table
CREATE TABLE reviews (
  id uuid DEFAULT gen_random_uuid() PRIMARY KEY,
  name text,
  email text NOT NULL,
  product_id uuid REFERENCES products(id) ON DELETE SET NULL,
  review_text text NOT NULL,
  rating integer CHECK (rating >= 1 AND rating <= 5),
  created_at timestamptz DEFAULT now(),
  UNIQUE(email)  -- one review per email
);

-- Enable RLS
ALTER TABLE products ENABLE ROW LEVEL SECURITY;
ALTER TABLE reviews ENABLE ROW LEVEL SECURITY;

-- Public read policies
CREATE POLICY "Public read products" ON products FOR SELECT USING (true);
CREATE POLICY "Public read reviews" ON reviews FOR SELECT USING (true);

-- Anon insert reviews
CREATE POLICY "Anon insert reviews" ON reviews FOR INSERT WITH CHECK (true);

-- Service role full access (admin)
CREATE POLICY "Admin all products" ON products USING (auth.role() = 'service_role');
CREATE POLICY "Admin all reviews" ON reviews USING (auth.role() = 'service_role');

2. Update Credentials

Open this HTML file and find the Supabase config at the bottom of the <script> section. Replace:

const SUPABASE_URL = 'YOUR_SUPABASE_URL';
const SUPABASE_ANON_KEY = 'YOUR_SUPABASE_ANON_KEY';

3. Admin Credentials

Default admin login: admin / wtf2024
Change the ADMIN_PASS constant in the script to update password.
