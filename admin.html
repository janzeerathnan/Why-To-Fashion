SUPABASE SETUP

1. Create Tables in Supabase SQL Editor

-- Products table
CREATE TABLE products (
  id uuid DEFAULT gen_random_uuid() PRIMARY KEY,
  name text NOT NULL,
  price numeric NOT NULL,
  image_url text,
  description text,
  category text,
  created_at timestamptz DEFAULT now()
);

-- Reviews table
CREATE TABLE reviews (
  id uuid DEFAULT gen_random_uuid() PRIMARY KEY,
  name text,
  email text NOT NULL,
  product_id uuid REFERENCES products(id) ON DELETE SET NULL,
  review_text text NOT NULL,
  rating integer CHECK (rating >= 1 AND rating <= 5),
  created_at timestamptz DEFAULT now(),
  UNIQUE(email)  -- one review per email
);

-- Enable RLS
ALTER TABLE products ENABLE ROW LEVEL SECURITY;
ALTER TABLE reviews ENABLE ROW LEVEL SECURITY;

-- Public read policies
CREATE POLICY "Public read products" ON products FOR SELECT USING (true);
CREATE POLICY "Public read reviews" ON reviews FOR SELECT USING (true);

-- Anon insert reviews
CREATE POLICY "Anon insert reviews" ON reviews FOR INSERT WITH CHECK (true);

-- Service role full access (admin)
CREATE POLICY "Admin all products" ON products USING (auth.role() = 'service_role');
CREATE POLICY "Admin all reviews" ON reviews USING (auth.role() = 'service_role');

2. Create Storage Bucket

- Go to Storage in Supabase dashboard

- Create a new bucket named product-images

- Mark the bucket as Public

- Add a Storage Policy allowing public reads and authenticated (or public) uploads

3. Update Credentials

Open app.js and find the Supabase config at the top. Replace:

const SUPABASE_URL = 'YOUR_SUPABASE_URL';
const SUPABASE_ANON_KEY = 'YOUR_SUPABASE_ANON_KEY';

4. Seed Sample Data

After creating the tables, run the seed to populate your database with sample products and reviews. Warning: this will add data — avoid running twice unless you clear the tables first.

First, run this SQL in your Supabase SQL Editor to allow anon inserts for seeding:

-- Allow anon to insert products (needed for seeding via admin panel)
CREATE POLICY "Anon insert products" ON products FOR INSERT WITH CHECK (true);
-- Allow anon to delete products (needed for admin delete)
CREATE POLICY "Anon delete products" ON products FOR DELETE USING (true);
-- Allow anon to update products (needed for admin edit)
CREATE POLICY "Anon update products" ON products FOR UPDATE USING (true);
-- Allow anon to delete reviews (needed for admin delete)
CREATE POLICY "Anon delete reviews" ON reviews FOR DELETE USING (true);

5. Admin Credentials

Default admin login: admin / wtf6
Change the ADMIN_PASS constant in app.js to update password.
